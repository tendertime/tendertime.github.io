docker强行关闭

是的如果你习惯了用ctrl+C结束进程你就会比我还蠢。
当然我现在更暴力我会另开一个docker stop xx然后两边都ctrl+D
当你犯下大错时，就会发现github和stackoverflow都没有解决方案。
是的端口冲突。关了进程还会再开；重写端口会出错；删了容器也好像幽灵一样。
这个端口已经被分配了，你不能用了。但是执着如我，对吧。
/etc/default底下没啥 etc/init.d底下也没啥 ubuntu根本就没有iptables
占端口的是docker-proxy proto 又不是外人，明摆着docker-daemon意外关闭的锅
删除/var/lib/docker/networks里的db文件之后世界清净了。
服务重启过，docker重装过，容器早就删空了 你说呢。
/var/lib/docker底下就那几个文件 想一想除了network别的好像也没这本事 索性清空 废了大不了重装。
success!
